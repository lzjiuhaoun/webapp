# 席位告警 (Seat Alert)

席位告警是 AAS-SIMP 平台对登录用户（席位）的行为进行实时风险判定的子领域。通过可配置的告警规则，在特定时间、特定行为维度上对席位进行分级告警，并支持黑白名单进行豁免或强制匹配。

## Language

**席位 (Seat)**:
平台上一个登录用户的操作位置。由用户名（全局唯一）标识，关联席位IP和席位状态。
_Avoid_: 用户、账号、终端、会话

**席位状态 (Seat Status)**:
席位当前的风险状态，二分法：离线 / 在线。在线状态下又根据是否触发告警规则细分为三级（黄色）、二级（橙色）、一级（红色）。
_Avoid_: 告警状态、席位等级、在线状态

**告警规则 (Alert Rule)**:
定义何时对席位产生告警的判定单元。包含基本信息（名称、日志类型）、规则类型（普通/组合）、匹配条件、关联黑白名单和告警等级。

**普通规则 (Normal Rule)**:
最基础的告警规则。包含匹配条件集合（字段-运算符-值）、关联黑白名单引用、直接配置告警等级。

**组合规则 (Composite Rule)**:
引用一条普通规则作为子规则，配置时间范围(分钟)和触发次数阈值。当子规则在指定时间范围内累计触发达到阈值时，组合规则被触发。

**黑白名单 (Access List)**:
一组用户名+账号的集合，可被多条告警规则引用。分为白名单和黑名单两种类别。

**白名单 (Whitelist)**:
告警规则引用白名单后，白名单中的席位在该规则下被完全豁免——即使匹配条件成立也不产生告警。
_Avoid_: 豁免名单、免告警名单

**黑名单 (Blacklist)**:
告警规则引用黑名单后，黑名单中的席位在原有匹配条件上 AND 追加匹配——强制纳入判定范围。
_Avoid_: 强制名单、必检名单

**匹配条件 (Matching Condition)**:
普通规则中的三元组：字段 + 运算符 + 值。多个条件之间通过"与"或"或"关联。
_Available Fields_: 文件名、发送方式、用户名称、用户账号、席位IP、登录时间、参与方

**告警等级 (Alert Level)**:
告警的严重程度分级。三级（黄色）最轻，一级（红色）最重。当多个规则同时命中同一席位时，取其最高等级作为最终席位状态。
_Avoid_: 告警级别、风险等级
