# 席位告警 (Seat Alert)

席位告警是 AAS-SIMP 平台对登录用户（席位）的行为进行实时风险判定的子领域。通过可配置的告警规则，在特定时间、特定行为维度上对席位进行分级告警，并支持黑白名单进行范围限定或豁免。

## Language

**席位 (Seat)**:
平台上一个登录用户的操作位置。由用户名（全局唯一）标识，关联席位IP。
_Avoid_: 用户、账号、终端、会话

**席位在线状态 (Seat Online Status)**:
席位当前的在线情况，二分法：离线 / 在线。离线状态下不展示告警状态。

**席位告警状态 (Seat Alert Status)**:
席位在在线状态下根据告警规则判定出的风险等级：正常、三级（黄色）、二级（橙色）、一级（红色）。多条规则同时命中时取最高等级。
_Avoid_: 席位等级、在线状态

**告警规则 (Alert Rule)**:
定义何时对席位产生告警的判定单元。包含基本信息（名称、日志来源）、规则类型（普通/组合）、匹配条件集合、关联黑白名单和告警等级。

**告警事件 (Alert Result)**:
规则引擎匹配成功时输出的持久化记录，存储在 `alert_result` 表中。普通规则命中一条产生一条告警事件（存单个审计 ID），组合规则命中产生一条告警事件（存逗号分隔的多个审计 ID 及实际触发次数）。所有展示数据通过审计源数据 ID 从 ES 查询，不冗余存储日志字段。
_Avoid_: 告警记录、告警流水

**日志来源 (Log Source)**:
告警规则匹配的日志来源分类，用于规则引擎筛选目标规则。供选值：`平台登录日志`、`IM登录日志`、`DLP日志`。
_Avoid_: 日志类型（此术语已用于操作类型）

**操作类型 (Log Type)**:
原始日志中记录的具体操作类别。对 IM 日志：登录、登出、私聊、群聊；对平台登录日志：登录、登出。标准化告警日志实体 `AlertRawLog` 中该字段名为 `logType`。
_Avoid_: 日志来源、日志分类

**发送方式 (Send Method)**:
IM 通讯中的消息投递方式，由操作类型派生：私聊 = 单发，群聊 = 群发。标准化告警日志实体 `AlertRawLog` 中该字段名为 `sendMethod`，由解析层在标准化时填充。
_Avoid_: 投递方式、消息类型

**告警事件源数据 (Alert Raw Log)**:
经过各日志源解析层标准化后的统一事件数据结构，作为告警规则引擎的输入。包含操作人信息、IP、时间戳、操作类型、发送方式、文件名、参与方、接收人列表等字段，以及 `rawData`（原始日志 JSON）和 `extensions` Map 用于未来扩展。字段明细见 [alert-raw-log-format.md](alert-raw-log-format.md)。
_Avoid_: 告警日志（易与告警流水混淆）

**普通规则 (Normal Rule)**:
基础的告警规则。包含匹配条件集合（字段-运算符-值，统一关联方式为全AND或全OR）、关联黑白名单引用、直接配置告警等级。

**组合规则 (Composite Rule)**:
引用一条普通规则作为子规则，配置时间范围（预留）和触发次数阈值。不同等级独立建成多条组合规则引用同一条普通规则。

**黑白名单 (Access List)**:
一组用户名+账号的集合，可被多条告警规则引用。分为白名单和黑名单两种类别。

**白名单 (Whitelist)**:
告警规则引用白名单且未配置黑名单后，白名单中的席位在该规则下被完全豁免——即使匹配条件成立也不产生告警。
_Avoid_: 豁免名单、免告警名单

**黑名单 (Blacklist)**:
告警规则引用黑名单后，黑名单定义触发范围——只有黑名单中的席位才会被该规则判定。黑名单优先级高于白名单：同时命中两份名单时，黑名单生效，白名单不豁免。
_Avoid_: 强制名单、必检名单

**匹配条件 (Matching Condition)**:
普通规则中的三元组：字段 + 运算符 + 值。多个条件之间统一通过"与"或"或"关联（不可混合）。字段不固定，按日志源动态配置。
_Available Operators_: 等于、不等于、大于等于、小于等于、包含、跨阵营

**跨阵营匹配 (Cross-Camp Matching)**:
匹配参与方（participatyParty）是否跨阵营通讯。值格式为阵营组定义，如 `红方|蓝方、白方|蓝方、绿方|红方|白方`。`|` 表示同阵营，`、` 分隔不同阵营组。发送方与接收方同属一个阵营组不告警，否则告警。

**告警等级 (Alert Level)**:
告警的严重程度分级。三级（黄色）最轻，一级（红色）最重。当多个规则同时命中同一席位时，取其最高等级作为最终席位告警状态。
_Avoid_: 告警级别、风险等级

## 黑白名单处理流程

```
用户命中匹配条件？
 ├── 否 → 不告警
 └── 是 → 配置了黑名单？
           ├── 是 → 用户在黑名单中？
           │         ├── 是 → 告警（白名单无效）
           │         └── 否 → 不告警
           └── 否 → 配置了白名单？
                     ├── 是 → 用户在白名单中？→ 是→豁免 / 否→告警
                     └── 否 → 告警
```

## 时间段规则设计

- 一条规则只对应一种时间段场景（工作/非工作），不同场景建不同规则
- 通过匹配条件中的"登录时间"字段配合 `大于等于` / `小于等于` 运算符实现
- 示例-工作时间：条件1=`登录时间 >= 08:00`，条件2=`登录时间 <= 19:00`，关联方式=与
