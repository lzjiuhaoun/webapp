# ACCEPTANCE.md — 端到端验收

## 验收场景

### S1: 审计日志采集与展示

**前置条件**：安全设备已配置 Syslog 指向平台，collection-engine 和 kafka-client 正常运行。

**步骤**：
1. 安全设备产生一条数据库操作审计日志
2. collection-engine 通过 UDP Syslog 接收日志
3. collection-engine 按 dataType 分类，发送到对应 Kafka Topic
4. kafka-client 消费 Kafka 消息，解析后写入 Elasticsearch 和 MySQL
5. web-app 提供检索接口，从 ES 查询日志记录
6. 用户通过 Web 界面查看审计日志详情

**通过标准**：
- [ ] Kafka Topic 中收到消息
- [ ] ES `aassimp-auditlog` 索引中可查询到该日志
- [ ] web-app 检索接口返回正确数据
- [ ] 日志字段解析完整（用户、IP、SQL、操作类型等）

### S2: 风险态势分析

**前置条件**：ES 中有足够的审计日志数据，report-engine 正常运行。

**步骤**：
1. report-engine 定时任务从 ES 聚合统计数据
2. 聚合结果写入 MySQL 报表表
3. 用户通过 web-app 访问风险态势页面
4. web-app 从 MySQL 查询报表数据
5. 前端展示风险计数、源 IP 分布、用户操作排行等

**通过标准**：
- [ ] report-engine 定时任务执行成功
- [ ] MySQL 报表表有统计数据
- [ ] web-app 态势接口返回数据
- [ ] 风险计数与实际数据匹配

### S3: 用户认证与权限

**前置条件**：CAS 服务端正常运行，web-app 已配置 CAS 地址。

**步骤**：
1. 用户访问 web-app HTTPS 地址
2. 重定向到 CAS 登录页面
3. 用户输入凭据完成认证
4. 返回 web-app，建立会话
5. 用户访问受保护资源，验证权限

**通过标准**：
- [ ] CAS SSO 认证通过
- [ ] 用户信息正确回传
- [ ] 会话有效期内无需重复登录
- [ ] 无权限资源访问被拒绝

### S4: 数据库资产发现

**前置条件**：web-app 正常运行，配置了扫描任务。

**步骤**：
1. 用户在 web-app 创建数据库发现任务
2. web-app 下发扫描指令
3. 扫描指定网段发现数据库实例
4. 发现结果录入资产列表
5. 用户查看资产详情

**通过标准**：
- [ ] 任务创建成功
- [ ] 扫描发现目标数据库
- [ ] 资产信息准确（IP、端口、类型、版本）
- [ ] 资产列表可正常展示

### S5: 进程异常恢复

**前置条件**：daemon 正常运行，Zookeeper 正常，各服务已注册。

**步骤**：
1. 模拟某个服务进程异常终止（如 collection-engine）
2. daemon 通过 ZK Watch 检测到节点消失
3. daemon 执行重启脚本恢复服务
4. 服务重新注册到 Zookeeper

**通过标准**：
- [ ] daemon 在检测到异常后及时触发重启
- [ ] 目标进程成功重启
- [ ] 重启后服务重新注册到 ZK
- [ ] 其他模块可恢复正常通信

### S6: 敏感数据发现与分类

**前置条件**：kafka-client 正常运行，ES 中有审计日志。

**步骤**：
1. 用户在 web-app 创建敏感数据扫描任务
2. kafka-client 的敏感数据匹配处理器执行扫描
3. 识别结果写入数据库
4. 用户查看敏感数据汇总

**通过标准**：
- [ ] 任务执行成功
- [ ] 敏感数据识别准确
- [ ] 分类结果正确
- [ ] 汇总页面数据完整

### S7: 备份与恢复

**前置条件**：web-app 正常运行，MySQL 数据库正常。

**步骤**：
1. 用户在 web-app 创建备份策略
2. 系统按策略执行全量/增量备份
3. 备份文件可选 SM4 加密
4. 用户从备份历史下载备份文件
5. 使用备份文件恢复数据库

**通过标准**：
- [ ] 备份策略创建成功
- [ ] 定时备份按策略执行
- [ ] 备份文件完整可下载
- [ ] 恢复后数据一致
