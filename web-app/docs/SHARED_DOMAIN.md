# SHARED_DOMAIN.md — 通用语言

## 核心领域对象定义

本文档定义 WebApp 系统共享的核心实体及其语义，确保所有模块对同一概念有统一认知。

**版本：** v1.0.0

---

## 1. 数据库资产（Database Asset）

| 属性 | 说明 |
|------|------|
| 定义 | 系统中纳管的目标数据库实例 |
| 生命周期 | 发现 → 注册 → 盘点 → 分类定级 → 监控 → 下线 |
| 核心属性 | IP地址、端口、数据库类型、库名、用户名、归属组织、物理位置 |
| 所属模块 | db-assets |
| 关联实体 | 安全设备（device）、脱敏策略（strategy）、敏感数据（sensdata） |

## 2. 安全设备（Security Device）

| 属性 | 说明 |
|------|------|
| 定义 | 接入系统的安全硬件/软件设备 |
| 设备类型 | CM（态势采集）、DM（数据库脱敏）、VS（数据库防火墙）、通用安全设备 |
| 核心属性 | 设备名称、IP地址、设备类型、通信端口、运行状态、告警信息 |
| 所属模块 | device |
| 关联实体 | 策略（strategy）、指令（instruction）、告警（system） |

## 3. 敏感数据（Sensitive Data）

| 属性 | 说明 |
|------|------|
| 定义 | 数据库或文件中包含的敏感信息字段或记录 |
| 敏感类型 | 身份证、手机号、银行卡号、姓名、地址、邮箱、IP地址等35+种 |
| 核心属性 | 数据来源（数据库/文件）、表名、字段名、敏感类型、匹配值、脱敏结果 |
| 所属模块 | sensdata |
| 关联实体 | 脱敏引擎（dm-engine）、脱敏策略（strategy）、数据库资产（db-assets） |

## 4. 脱敏策略（Desensitization Strategy）

| 属性 | 说明 |
|------|------|
| 定义 | 对敏感数据进行脱敏处理的规则和配置 |
| 脱敏方式 | 映射、随机、降维、可逆、分区 |
| 核心属性 | 策略名称、目标设备、目标数据库、脱敏算法、敏感类型、生效状态 |
| 所属模块 | strategy |
| 关联实体 | 脱敏引擎（dm-engine）、安全设备（device）、敏感数据（sensdata） |

## 5. 风险（Risk）

| 属性 | 说明 |
|------|------|
| 定义 | 系统识别的安全风险事件或趋势 |
| 风险分类 | 攻击风险、漏洞风险、运维风险、数据访问风险、总体风险 |
| 风险等级 | 高、中、低 |
| 核心属性 | 风险类型、风险等级、涉及资产、发生时间、详情描述 |
| 所属模块 | risk |
| 关联实体 | 态势（situation）、数据库资产（db-assets）、安全设备（device） |

## 6. 态势（Situation）

| 属性 | 说明 |
|------|------|
| 定义 | 基于多维数据聚合的安全态势展示 |
| 态势维度 | 总体态势、风险态势、健康态势、漏洞态势、数据流转 |
| 核心属性 | 统计指标、趋势数据、排名数据、风险分布 |
| 所属模块 | situation |
| 关联实体 | 风险（risk）、数据库资产（db-assets）、安全设备（device）、敏感数据（sensdata） |

## 7. 审计策略（Audit Strategy）

| 属性 | 说明 |
|------|------|
| 定义 | 对数据库操作行为进行审计的规则 |
| 核心属性 | 策略名称、审计对象、审计条件、触发动作、告警级别 |
| 所属模块 | audit |
| 关联实体 | 数据库资产（db-assets）、安全设备（device） |

## 8. 水印规则（Watermark Rule）

| 属性 | 说明 |
|------|------|
| 定义 | 对数据库查询结果添加水印以防止数据泄露的规则 |
| 水印算法 | 多种算法（参见 `WatermarkAlgorithm` 枚举） |
| 核心属性 | 规则名称、目标数据库、水印算法、水印内容、生效条件 |
| 所属模块 | strategy |

## 9. 告警（Alarm）

| 属性 | 说明 |
|------|------|
| 定义 | 系统检测到的安全事件或异常状态 |
| 告警级别 | 紧急、重要、一般、提示 |
| 通知渠道 | 邮件、短信、钉钉、SNMP、Syslog |
| 核心属性 | 告警来源、告警级别、告警内容、通知状态、处置状态 |
| 所属模块 | system（系统告警管理）、各模块产生告警 |
| 关联实体 | 通知渠道（server）、安全设备（device） |

## 10. 组织与用户（Organization & User）

| 属性 | 说明 |
|------|------|
| 定义 | 系统的管理组织架构和用户权限体系 |
| 核心属性 | 组织树、逻辑区域、物理位置、用户角色、权限模块 |
| 所属模块 | auth（用户）、config（组织） |
| 关联实体 | 数据库资产（db-assets）、安全设备（device） |

## 11. 指令（Instruction）

| 属性 | 说明 |
|------|------|
| 定义 | 向安全设备下发的操作指令 |
| 指令类型 | 脱敏指令、水印指令、策略同步、配置更新等 |
| 指令状态 | 待发送、已发送、已确认、失败、超时 |
| 核心属性 | 目标设备、指令类型、载荷数据、确认状态、发送时间 |
| 所属模块 | instruction |
| 关联实体 | 安全设备（device）、策略（strategy） |

## 12. 文件数据源（File Data Source）

| 属性 | 说明 |
|------|------|
| 定义 | 系统纳管的非数据库型数据源（文件） |
| 核心属性 | 文件路径、目录结构、模板配置、扫描周期 |
| 所属模块 | filedata |
| 关联实体 | 敏感数据（sensdata） |

## 13. 备份（Backup）

| 属性 | 说明 |
|------|------|
| 定义 | 系统配置和数据的备份管理 |
| 备份类型 | 全量备份、增量备份 |
| 核心属性 | 备份策略、备份频率、备份路径、备份历史、恢复状态 |
| 所属模块 | backup |
